# バックエンド (通信制御, システム)

言語: Node.js + JavaScript
フレームワーク: Express

## パッケージ
- express
- ws
- nodemon (開発用)

- ゲームのシステム (座標管理など)
- 通信制御

join時に送られたuuidをもとに、スマートフォンのクライアントを識別する。
WebSocketで実装する。クライアント(スマートフォン)からのメッセージを受け取って、ゲームのシステムに反映させる。
ゲームのシステムは、クライアント(スマートフォン)からのメッセージを受け取って、キャラクターの座標を更新したり、射撃の処理を行ったりする。

横移動のみで、縦移動はなし。

敵は自動で動く。敵の座標もフロントエンドに送信する。
射撃の弾や敵、プレイヤーなどすべてのオブジェクトを管理する

## フロントエンドに送信するデータ
### キャラクターの座標
```json
{
  "type": "update",
  "characters": [
    {
      "id": "<端末のuuid>",
      "x": 3
    },
    {
      "id": "<端末のuuid>",
      "x": -2
    }
  ],
  "enemies": [
    {
      "id": "<敵のuuid>",
      "x": 1,
      "y": 2
    },
    {
      "id": "<敵のuuid>",
      "x": -1,
      "y": 3
    }
  ],
  "bullets": [
    {
      "id": "<弾のuuid>",
      "x": 0,
      "y": 1
    },
    {
      "id": "<弾のuuid>",
      "x": 2,
      "y": 3
    }
  ]
}
```



## スマートフォンから受信するデータ
### 参加
```json
{
  "type": "join",
  "id": "<端末のuuid>"
}
```

### 離脱
```json
{
  "type": "leave"
}
```


### 移動
```json
{
  "type": "move",
  "delta": -0.4
}
```

スマートフォンのジャイロセンサーからdelta (移動量)を求める. 
範囲は-1 ～ 1

### 射撃
```json
{
  "type": "shoot"
}
```
