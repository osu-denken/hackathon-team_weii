# バックエンド (通信制御, システム)

言語: Node.js + JavaScript
フレームワーク: Express

## ngrokで公開する
1. ngrokのCLIをインストールしてログイン
2. サーバー起動
3. ngrokでトンネルを作成

```bash
npm run dev
```

```bash
npm run ngrok
```

## パッケージ
- express
- ws
- nodemon (開発用)

- ゲームのシステム (座標管理など)
- 通信制御

join時に送られたuuidをもとに、スマートフォンのクライアントを識別する。
WebSocketで実装する。クライアント(スマートフォン)からのメッセージを受け取って、ゲームのシステムに反映させる。
ゲームのシステムは、クライアント(スマートフォン)からのメッセージを受け取って、キャラクターの座標を更新したり、射撃の処理を行ったりする。

横移動のみで、縦移動はなし。

敵は自動で動く。敵の座標もフロントエンドに送信する。
射撃の弾や敵、プレイヤーなどすべてのオブジェクトを管理する

## フロントエンドに送信するデータ
### キャラクターの座標
```json
{
  "type": "update",
  "players": [
    {
      "id": "<端末のuuid>",
      "x": 3
    },
    {
      "id": "<端末のuuid>",
      "x": -2
    }
  ],
  "enemies": [
    {
      "id": "<敵のuuid>",
      "x": 1,
      "y": 2
    },
    {
      "id": "<敵のuuid>",
      "x": -1,
      "y": 3
    }
  ],
  "bullets": [
    {
      "id": "<弾のuuid>",
      "x": 0,
      "y": 1
    },
    {
      "id": "<弾のuuid>",
      "x": 2,
      "y": 3
    }
  ]
}
```

### 差分更新 (Delta Updates)
通信量削減のため、通常時は前回の送信内容と比較して**変化のあったデータ（差分）のみ**を送信します（要素に変化が一切ない場合は送信自体をスキップします）。
差分データとして送信される場合、ペイロードには `"isDelta": true` が付与されます。クライアント側は受信したデータを既存のStateにマージする形で処理します。

```json
{
  "type": "update",
  "isDelta": true,
  "game": {
    "timeRemainingMs": 9950
  }
}
```

バックエンド側の送信メソッド (`sendState` や `sendStateToPlayers`) には `isDelta` 引数（デフォルト: `true`）が用意されています。
- `sendState(true)` または引数省略: 差分（Delta）があれば送信する。
- `sendState(false)`: 差分に関わらず、強制的にすべての状態をフルで送信する（初期接続時やゲームリセット時、難易度変更時などに利用）。

## スマートフォンから受信するデータ
### 参加
```json
{
  "type": "join",
  "id": "<端末のuuid>"
}
```

### 離脱
```json
{

  "type": "leave"
}
```


### 移動
```json
{
  "type": "move",
  "delta": -0.4
}
```

スマートフォンのジャイロセンサーからdelta (移動量)を求める. 
範囲は-1 ～ 1

### 射撃
```json
{
  "type": "shoot"
}
```
