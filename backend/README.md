# バックエンド (通信制御, システム)

言語: Node.js + JavaScript
フレームワーク: Express

## ngrokで公開する
1. ngrokのCLIをインストールしてログイン
2. サーバー起動
3. ngrokでトンネルを作成

```bash
npm run dev
```

```bash
npm run ngrok
```

## パッケージ
- express
- ws
- nodemon (開発用)

- ゲームのシステム (座標管理など)
- 通信制御

join時に送られたuuidをもとに、スマートフォンのクライアントを識別する。
WebSocketで実装する。クライアント(スマートフォン)からのメッセージを受け取って、ゲームのシステムに反映させる。
ゲームのシステムは、クライアント(スマートフォン)からのメッセージを受け取って、キャラクターの座標を更新したり、射撃の処理を行ったりする。

横移動のみで、縦移動はなし。

敵は自動で動く。敵の座標もフロントエンドに送信する。
射撃の弾や敵、プレイヤーなどすべてのオブジェクトを管理する

### 差分更新 (Delta Updates)
通信量削減のため、通常時は前回の送信内容と比較して**変化のあったデータ（差分）のみ**を送信します（要素に変化が一切ない場合は送信自体をスキップします）。
差分データとして送信される場合、ペイロードには `"isDelta": true` が付与されます。クライアント側は受信したデータを既存のStateにマージする形で処理します。

```json
{
  "type": "update",
  "isDelta": true,
  "game": {
    "timeRemainingMs": 9950
  }
}
```

バックエンド側の送信メソッド (`sendState` や `sendStateToPlayers`) には `isDelta` 引数（デフォルト: `true`）が用意されています。
- `sendState(true)` または引数省略: 差分（Delta）があれば送信する。
- `sendState(false)`: 差分に関わらず、強制的にすべての状態をフルで送信する（初期接続時やゲームリセット時、難易度変更時などに利用）。

### 推測描画 (Client-Side Prediction)
弾丸・敵・アイテムなどの予測可能なエンティティについては、毎フレームの座標(`x`, `y`)を送信しません。
代わりにサーバーはエンティティの生成時（または同期時）に、「初期座標(`startX`, `startY`)」「生成時刻(`createdAt`)」に加え、「X座標の計算式(`fx`)」「Y座標の計算式(`fy`)」の文字列を送信します（これらのエンティティには `isPredictable: true` が付与されます）。
フロントエンド側は、受信した数式文字列を `new Function` でコンパイルし、サーバー時刻(`game.serverNow`)とのオフセットを用いた経過時間 `t` を代入することで、クライアントの滑らかなリフレッシュレートで現在位置を計算・描画します。

**同期ズレ補正 (Resync)**
`backend/src/constants/gameConfig.js` の `RESYNC_INTERVAL_MS`（デフォルト2000ms）ごとに、サーバーから最新の座標が `startX`, `startY` として送信され、基準時刻がリセットされることで定期的に同期ズレを補正します。
また、`ENABLE_PREDICTION` フラグでこの推測描画機能自体のオンオフを一括で制御できます。


## スマートフォンから受信するデータ
### 参加
```json
{
  "type": "join",
  "id": "<端末のuuid>"
}
```

### 離脱
```json
{

  "type": "leave"
}
```


### 移動
```json
{
  "type": "move",
  "delta": -0.4
}
```

スマートフォンのジャイロセンサーからdelta (移動量)を求める. 
範囲は-1 ～ 1

### 射撃
```json
{
  "type": "shoot"
}
```
